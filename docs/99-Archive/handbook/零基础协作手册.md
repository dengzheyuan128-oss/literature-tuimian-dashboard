## 准备工作

### 你需要的工具

#### 1. GitHub Desktop（推荐） 或 Git 命令行
- **GitHub Desktop 下载**: https://desktop.github.com/
- 这是一个图形界面工具，不需要记命令

#### 2. VS Code（或任何文本编辑器）
- **VS Code 下载**: https://code.visualstudio.com/
- 用来编辑代码文件

#### 3. Node.js 和 pnpm
- **检查是否已安装**: 
  - Windows: 打开"命令提示符"（按 Win+R，输入 cmd，回车）
  - Mac: 打开"终端"（Command+Space，输入 terminal）
  - 输入: `node --version`
  - 如果显示版本号（如 v18.x.x），说明已安装
  - 如果提示"命令不存在"，需要先安装

### 开始前的检查清单

```markdown
检查清单（请逐项确认）:
□ GitHub Desktop 已安装并登录
□ VS Code 已安装
□ 项目已在本地克隆（在 GitHub Desktop 中能看到）
□ 能运行 pnpm dev（项目能在浏览器打开）
□ GitHub Issue #1 已创建
```

## Task 1: 修复空壳卡片问题

### 🎯 目标
让所有不完整的院校信息都显示"数据补充中"的提示，而不是空白一片。

### 📋 你需要做的事情（分步详解）

---

#### Step 1: 获取项目当前状态（5分钟）

**目的**: 让 Claude 知道你的项目现在是什么样子

**操作步骤**:

**1.1 打开 GitHub Desktop**
- 在左上角，确认当前仓库是 `literature-tuimian-dashboard`
- 如果不是，点击 "Current Repository" 切换

**1.2 查看当前分支**
- 看界面顶部的 "Current Branch"
- 记下是 `main` 还是其他名字

**1.3 检查是否有未保存的改动**
- 看左侧边栏 "Changes" 标签
- 如果有文件列表，说明有未保存的改动
- 如果是空的，说明当前很干净（这是最好的）

**1.4 创建新的工作分支（重要！）**

为什么要创建新分支？
- 就像给文档做了一个"副本"
- 改坏了可以随时丢弃
- 改好了再合并回主版本

**操作**:
1. 点击顶部的 "Current Branch"
2. 点击 "New Branch"
3. 在弹出框中输入分支名: `phase-0-fix-empty-cards`
4. 点击 "Create Branch"

**1.5 把信息告诉 Claude**

请到 GitHub Issue #1 下方评论区，复制粘贴以下内容：

```markdown
### 📊 项目当前状态

✅ 准备工作完成！

**环境信息**:
- 当前分支: [从 GitHub Desktop 看到的分支名]
- 本地是否有未提交改动: 是 / 否
- pnpm dev 能否正常运行: 能 / 不能（如果不能，说明具体错误）

**准备开始 Task 1**: 修复空壳卡片

@Claude 我已经准备好了，请给我第一批文件！
```

我会在 Issue 评论区回复，提供：

**你需要做什么**:

#### Step 3: 下载并应用文件（10分钟）

**Claude 回复后，你会看到类似这样的内容**:

```markdown
### 📦 文件已生成！

请按以下步骤操作：

**文件 1: Home.tsx**
[点击这里下载](链接) 或复制下方代码
```

**3.1 下载文件的两种方式**

**方式A: 直接复制代码（推荐）**
1. 在 Claude 的回复中，找到代码块（通常有 ```typescript 标记）
2. 点击代码块右上角的"复制"图标
3. 代码已复制到剪贴板

**方式B: 下载文件**
1. 点击 Claude 提供的下载链接
2. 文件会下载到你的"下载"文件夹
3. 记住文件名（如 `Home.tsx`）

**3.2 找到要修改的文件位置**

使用 VS Code:
1. 打开 VS Code
2. 点击 "File" → "Open Folder"
3. 选择你的项目文件夹（`literature-tuimian-dashboard`）
4. 在左侧文件树中，按这个路径找：
   ```
   literature-tuimian-dashboard
   └── client
       └── src
           └── pages
               └── Home.tsx  ← 就是这个文件
   ```

**3.3 替换文件内容**

**重要：先备份！**
1. 右键点击 `Home.tsx`
2. 选择 "Copy"
3. 在同一个文件夹右键 → "Paste"
4. 会生成一个 `Home copy.tsx`（这就是备份）

**然后替换**:
1. 双击打开 `Home.tsx`
2. 按 `Ctrl+A`（Mac: `Command+A`）全选所有内容
3. 按 `Delete` 删除
4. 按 `Ctrl+V`（Mac: `Command+V`）粘贴 Claude 提供的新代码
5. 按 `Ctrl+S`（Mac: `Command+S`）保存

**3.4 检查是否粘贴成功**

看文件第一行应该是这样的：
```typescript
import { useState, useMemo } from "react";
```

如果是，说明成功！如果不是，重新粘贴一次。

#### Step 4: 测试改动（5分钟）

**4.1 启动开发服务器**

1. 打开终端（命令提示符）
2. 进入项目文件夹：
   ```bash
   cd 你的项目路径/literature-tuimian-dashboard
   # 例如: cd C:\Users\YourName\Documents\literature-tuimian-dashboard
   ```
3. 运行命令：
   ```bash
   pnpm dev
   ```
4. 等待启动，看到类似这样的提示：
   ```
   ➜  Local:   http://localhost:5173/
   ```

**4.2 在浏览器中检查**

1. 打开浏览器
2. 访问 `http://localhost:5173`
3. **重点检查这些**:
   - □ 页面能正常加载（不是白屏）
   - □ 院校卡片中，如果信息不完整，是否显示"数据补充中"或"部分信息"标记
   - □ 没有完全空白的卡片
   - □ 控制台没有红色错误（按 F12 打开控制台）

**4.3 截图保存证据**

1. 在浏览器中按 `Win+Shift+S`（Mac: `Command+Shift+4`）
2. 截取页面效果
3. 保存截图

**4.4 反馈结果给 Claude**

在 GitHub Issue #1 评论：

```markdown
### ✅ Task 1 测试结果

**测试时间**: [当前日期时间]

**测试结果**: 
- pnpm dev 启动: ✅ 成功 / ❌ 失败
- 页面加载: ✅ 正常 / ❌ 白屏
- 空壳卡片: ✅ 已修复，都有提示 / ❌ 还有空白

**截图**:
[粘贴截图]

**遇到的问题**（如果有）:
[描述问题]

**结论**: 可以进入下一步 / 需要调整
```

#### Step 5: 提交改动（5分钟）

**只有在测试通过后才执行这步！**

**5.1 在 GitHub Desktop 中查看改动**

1. 打开 GitHub Desktop
2. 看左侧 "Changes" 标签
3. 应该能看到 `Home.tsx` 被修改了

**5.2 写提交说明**

- Description（描述，可选）:

**5.3 提交**

1. 点击左下角蓝色按钮 "Commit to phase-0-fix-empty-cards"

**5.4 确认提交成功**

1. 打开浏览器
2. 访问 `https://github.com/你的用户名/literature-tuimian-dashboard/tree/phase-0-fix-empty-cards`

## 常见问题解答

### ❓ 如果 `pnpm dev` 启动失败怎么办？

**常见原因和解决方法**:

**错误1: `command not found: pnpm`**
```bash
# 解决方法：安装 pnpm
npm install -g pnpm
```

**错误2: `Port 5173 is already in use`**
```bash
# 解决方法：换个端口
pnpm dev --port 3000
```

**错误3: 代码有语法错误**
```
# 解决方法：检查是否完整复制了 Claude 的代码
# 或者在 Issue 中告诉 Claude 具体的错误信息
```

---

### ❓ 如果改错了想恢复怎么办？

**方法1: 使用备份文件**
```
1. 找到之前复制的 "Home copy.tsx"
2. 删除现在的 "Home.tsx"
3. 把 "Home copy.tsx" 重命名为 "Home.tsx"
```

**方法2: 使用 Git 恢复**
```bash
# 在 GitHub Desktop 中
1. 右键点击改动的文件
2. 选择 "Discard Changes"
3. 文件会恢复到修改前的状态
```

**方法3: 删除分支重新开始**
```bash
# 在 GitHub Desktop 中
1. 切换回 main 分支（点击 Current Branch → main）
2. 右键点击 phase-0-fix-empty-cards 分支
3. 选择 "Delete"
4. 重新创建分支，从头开始
```

### ❓ GitHub Desktop 和命令行有什么区别？

**GitHub Desktop（推荐新手）**:
- ✅ 图形界面，直观
- ✅ 不需要记命令
- ✅ 能看到改动的可视化对比
- ❌ 功能相对简单

**命令行（适合进阶）**:
- ✅ 功能强大
- ✅ 可以自动化
- ❌ 需要记忆命令
- ❌ 出错时不好排查

**建议**: 先用 GitHub Desktop，熟练后再学命令行

---

### ❓ 什么时候可以合并到 main 分支？

**合并的条件**:
1. ✅ Task 1 和 Task 2 都完成
2. ✅ 本地测试全部通过
3. ✅ 在 Vercel Preview 中也验证过（如果有）
4. ✅ Claude 确认无问题

**合并操作**（在 GitHub 网页上）:
1. 访问你的仓库
2. 点击 "Pull requests"
3. 点击 "New pull request"
4. 选择 `phase-0-fix-empty-cards` → `main`
5. 点击 "Create pull request"
6. 填写标题: "Phase 0: 修复空壳卡片和添加版本信息"
7. 点击 "Create pull request"
8. 检查 CI 是否通过（如果配置了）
9. 点击 "Merge pull request"
