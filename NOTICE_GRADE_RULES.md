# 链接质量分级规则

**文档版本**: v1.0  
**适用Schema**: v1.1  
**最后更新**: 2026-01-16

---

## 1. 文档定位

本文件用于回答一个核心问题：

> **在推免信息场景下，什么样的链接"值得信任到可以直接给学生用"？**

结论是：
- 不是"能打开"
- 不是"官网"
- 而是：**是否能稳定、直接、明确指向当年推免决策信息**

---

## 2. 链接等级定义（A / B / C / D）

### A 级｜官方推免通知页（最高可信）

**定义**：
- 官方网站发布的
- 明确针对"推免 / 接收推免 / 推荐免试"的
- **具体通知页面**

**必须满足**：
- 页面标题或正文中出现：
  - "推免 / 推荐免试 / 接收推免"
- 页面内容可直接判断：
  - 报名条件
  - 时间节点
  - 适用学院或专业

**典型示例**：
- 研究生院发布的《××大学 2026 年接收推荐免试研究生通知》
- 文学院发布的《××学院关于接收 2026 年推免生的通知》

**反例（即使是官网，也不得评 A）**：
- 只有"招生信息汇总"但未明确推免
- 往年推免通知，年份未核验

---

### B 级｜官方通知列表页 / 推免栏目页（可接受）

**定义**：
- 官方网站上的栏目页
- 能稳定定位到"推免 / 接收推免"相关内容
- 但不是单条通知正文

**适用场景**：
- 当年推免通知尚未发布
- 学校习惯将推免信息集中在栏目中

**允许条件**：
- 页面结构稳定
- 用户可以通过 1 次点击或滚动找到推免相关信息

**重要说明**：
> B 级在**非头部学校**是长期可接受状态，不要求强制升级为 A。

---

### C 级｜学院 / 研究生院首页（不推荐）

**定义**：
- 官网首页
- 未直接指向推免相关内容

**问题**：
- 用户需要自行搜索
- 信息定位成本高

**处理原则**：
- 视为"兜底链接"
- 应在后续版本中优先替换

---

### D 级｜第三方平台 / 聚合站（不可接受）

**定义**：
- 非学校官方域名
- 内容转载、聚合或镜像

**风险**：
- 信息可能过期
- 页面结构不可控
- 随时失效

**处理原则**：
> D 级链接 **必须清零**，不得作为正式数据发布。

---

## 3. 年份（Year）与可信度的关系

### 3.1 核心原则

> **年份未核验的通知，不得进入最高可信状态。**

即使满足 A 级页面条件，只要年份无法确认：
- `linkGrade` 最高只能是 **B**

---

### 3.2 yearStatus 定义

| yearStatus | 含义 |
|-----------|------|
| verified | 已确认与 notice.year 一致 |
| unverified | 暂未确认（常见于新一轮尚未发布） |
| mismatch | 与 notice.year 明显不一致 |

---

### 3.3 判定规则（人工与自动共用）

- 若 URL / 页面标题 / requirements_raw 中出现 `notice.year`
  → `yearStatus = verified`
- 若未出现，但无明确冲突
  → `yearStatus = unverified`
- 若明确出现其他年份
  → `yearStatus = mismatch`

---

## 4. 常见误区与强制降级规则

### 误区 1：官网 + 正式页面 = A

❌ 错误。

- 官网的"招生简章""往年通知"
- 若年份未确认
- **不得评 A**

---

### 误区 2：列表页一定不如正文

❌ 错误。

- 稳定、结构清晰的官方栏目页
- 在推免季前期往往比"过期正文页"更可靠

---

### 误区 3：为了健康度，把所有 B 强行升级为 A

❌ 错误。

- 健康度是"风险控制指标"
- 不是"装饰性 KPI"

---

## 5. 修复流程建议（人工）

1. 优先查找：研究生院 → 文学院 → 中文系
2. 若当年未发布：
   - 保留最近一次官方通知
   - 标记 `yearStatus = unverified`
   - `linkGrade ≤ B`
3. 严禁使用第三方平台作为最终链接

## 6. 自动检测规则（check-data-quality.js）

### 6.1 Year 校验逻辑

```javascript
function verifyYear(notice) {
  const year = notice.year;
  const evidenceText = [
    notice.url,
    notice.title || '',
    notice.requirements_raw || ''
  ].join(' ');

  if (evidenceText.includes(String(year))) {
    return { yearStatus: 'verified' };
  }

  const otherYearMatch = evidenceText.match(/20\d{2}/g);
  if (otherYearMatch && !otherYearMatch.includes(String(year))) {
    return { yearStatus: 'mismatch' };
  }

  return { yearStatus: 'unverified' };
}
```

### 6.2 Grade 降级保护逻辑

```javascript
function enforceGradePolicy(notice) {
  if (notice.yearStatus !== 'verified' && notice.linkGrade === 'A') {
    notice.linkGrade = 'B';
    notice.warnings.push('Year not verified: downgrade A → B');
  }
}
```

## 7. 数据健康度指标

### 7.1 定义

**数据健康度** = (A级 + B级) / 总数 × 100%

### 7.2 目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 整体健康度 | ≥ 90% | A+B级占比 |
| D级链接 | 0 | 必须清零 |
| 第一梯队A级 | 100% | 或合理范围内 |

### 7.3 分级标准

- **优秀**: ≥ 95%
- **良好**: 90-94%
- **及格**: 85-89%
- **不及格**: < 85%

## 8. 维护原则

### 8.1 优先级

1. **P0**: D级链接清零
2. **P1**: 第一梯队A级化
3. **P2**: 第二梯队B级稳定
4. **P3**: 第三梯队C级替换

### 8.2 更新频率

- **推免季前**（每年8-9月）：全量检查
- **推免季中**（每年9-10月）：高频监控
- **推免季后**（每年11月-次年7月）：低频维护

### 8.3 人工接管触发条件

满足任一即触发：

- 搜索超过 3 次
- 出现多个疑似但无法确认的链接
- 官网结构变化

→ **必须停止自动搜索，提交待办给维护者**
